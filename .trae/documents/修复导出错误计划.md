# 修复导出错误计划

## 问题描述
浏览器控制台报错：
```
Uncaught SyntaxError: The requested module '/src/api/auth.ts?t=1772887811180' does not provide an export named 'User'
```

## 问题分析

这个错误说明有文件尝试从 `/src/api/auth.ts` 导入 `User` 类型，但是该文件没有导出 `User`。

### 可能的导入位置
1. `src/pages/Login.tsx` - 可能导入了 User
2. `src/pages/Profile.tsx` - 可能导入了 User
3. `src/pages/Home.tsx` - 可能导入了 User
4. `src/store/authStore.ts` - 可能导入了 User
5. 其他组件文件

### 问题原因
在 `src/api/auth.ts` 中，`User` 是一个 TypeScript 接口（interface），需要使用 `export interface` 正确导出。

## 诊断步骤

### 步骤 1：检查 auth.ts 导出
查看 `src/api/auth.ts` 文件，确认是否正确导出 `User` 接口。

### 步骤 2：查找所有导入 User 的文件
搜索代码库中所有从 auth.ts 导入 User 的地方。

### 步骤 3：检查导入语法
确认导入语法是否正确：
```typescript
// 正确的导入方式
import { User } from '../api/auth';

// 或者类型导入
import type { User } from '../api/auth';
```

## 修复方案

### 方案 A：确保正确导出（推荐）

在 `src/api/auth.ts` 中确认：
```typescript
// ✅ 正确 - 导出接口
export interface User {
  id: string;
  username: string;
  // ...
}

// ✅ 正确 - 导出类型
export type User = {
  id: string;
  username: string;
  // ...
};
```

### 方案 B：使用类型导入

如果 User 只是类型，使用类型导入：
```typescript
import type { User } from '../api/auth';
```

### 方案 C：检查循环依赖

如果存在循环依赖，可能需要重构代码结构。
