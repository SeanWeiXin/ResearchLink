
**注意事项**：
- ⚠️ 确保 `.env` 文件没有被提交到 Git
- ⚠️ 检查 `uploads/` 目录是否在 `.gitignore` 中

## 🗄️ 第二阶段：部署 MongoDB Atlas 数据库（约 20 分钟）

### 步骤 2.1：注册 MongoDB Atlas

1. 访问 https://cloud.mongodb.com
2. 点击 "Sign Up" 注册账号（可使用 GitHub/Google 账号快捷登录）
3. 验证邮箱地址

### 步骤 2.2：创建免费集群

1. 登录后，点击 "Create a cluster"
2. 选择配置：
   - **Cluster Tier**: 选择 **M0 FREE**（免费层）
   - **Cloud Provider**: 选择 **AWS**
   - **Region**: 选择 **Asia Pacific (Singapore)** 或 **Tokyo**（延迟较低）
   - **Cluster Name**: `researchlink-cluster`
3. 点击 "Create Cluster"（创建需要几分钟）

### 步骤 2.3：配置数据库用户

1. 点击左侧菜单 "Database Access"
2. 点击 "+ ADD NEW DATABASE USER"
3. 配置用户：
   - **Authentication Method**: Password
   - **Username**: `researchlink_user`
   - **Password**: 点击 "Autogenerate Secure Password" 并复制保存
   - **Database User Privileges**: 选择 "Read and write to any database"
4. 点击 "Add User"

### 步骤 2.4：配置网络访问

1. 点击左侧菜单 "Network Access"
2. 点击 "+ ADD IP ADDRESS"
3. 选择：
   - 点击 "ALLOW ACCESS FROM ANYWHERE"（添加 0.0.0.0/0）
   - 或者添加你的本地 IP（更安全）
4. 点击 "Confirm"

### 步骤 2.5：获取连接字符串

1. 点击左侧菜单 "Database"
2. 在集群卡片上点击 "Connect"
3. 选择 "Connect your application"
4. 选择驱动：
   - **Driver**: Node.js
   - **Version**: 5.5 or later
5. 复制连接字符串，格式类似：
   ```
   mongodb+srv://researchlink_user:<password>@researchlink-cluster.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
6. **重要**：将 `<password>` 替换为你刚才设置的密码

### 步骤 2.6：保存数据库配置信息

将以下信息保存到安全的地方（不要提交到 Git）：

```
MongoDB 连接字符串：mongodb+srv://researchlink_user:你的密码@researchlink-cluster.xxxxx.mongodb.net/researchlink?retryWrites=true&w=majority
数据库用户名：researchlink_user
数据库密码：你的密码
```

## 🚀 第三阶段：部署后端服务到 Render（约 20 分钟）

### 步骤 3.1：注册 Render

1. 访问 https://render.com
2. 点击 "Get Started for Free"
3. 使用 GitHub 账号登录（推荐）或邮箱注册

### 步骤 3.2：创建 Web Service

1. 登录后，点击 Dashboard 中的 "New +"
2. 选择 "Web Service"
3. 连接 GitHub 仓库：
   - 点击 "Connect account" 授权 Render 访问 GitHub
   - 找到你的 `ResearchLink` 仓库并连接

### 步骤 3.3：配置 Web Service

填写以下配置：

**Basic Settings**:
- **Name**: `researchlink-api`（会自动生成 URL）
- **Region**: `Singapore`（新加坡，离中国较近）
- **Branch**: `main`
- **Root Directory**: `server`（重要！指定后端目录）
- **Runtime**: `Node`
- **Build Command**: `npm install`
- **Start Command**: `npm start`
- **Instance Type**: `Free`（免费层）

**Environment Variables**（点击 "Advanced" 展开）:
点击 "Add Environment Variable" 添加以下变量：

```
NODE_ENV = production
MONGODB_URI = mongodb+srv://researchlink_user:你的密码@researchlink-cluster.xxxxx.mongodb.net/researchlink?retryWrites=true&w=majority
JWT_SECRET = 生成一个随机字符串（见下方说明）
MAX_FILE_SIZE = 52428800
UPLOAD_PATH = ./uploads
CORS_ORIGIN = https://researchlink.vercel.app（先留空，部署前端后填写）
PORT = 5000
```

**生成 JWT_SECRET 的方法**：
```bash
# 在 PowerShell 中执行
[node]::System.Guid.NewGuid().ToString()
```

### 步骤 3.4：配置 CORS

在开始部署前，需要确保后端代码支持 CORS 配置。

检查 `server/server.js` 中是否有以下配置：

```javascript
const cors = require('cors');

// 使用环境变量配置 CORS
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'http://localhost:5173',
  credentials: true
}));
```

如果没有，需要先修改代码（后续我会帮你修改）。

### 步骤 3.5：开始部署

1. 点击 "Create Web Service"
2. 等待部署完成（约 3-5 分钟）
3. 部署成功后，你会得到一个 URL，格式类似：
   ```
   https://researchlink-api.onrender.com
   ```
4. 保存这个 URL

### 步骤 3.6：测试后端 API

在浏览器中访问：
```
https://researchlink-api.onrender.com/api
```

或者使用 curl 测试：
```bash
curl https://researchlink-api.onrender.com/api
```

## 🎨 第四阶段：部署前端服务到 Vercel（约 15 分钟）

### 步骤 4.1：注册 Vercel

1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 使用 GitHub 账号登录（推荐）

### 步骤 4.2：导入项目

1. 登录后，点击 "Add New..." -> "Project"
2. 在 "Import Git Repository" 中找到你的 `ResearchLink` 仓库
3. 点击 "Import"

### 步骤 4.3：配置项目

**Configure Project**:

- **Project Name**: `researchlink`（会影响域名）
- **Framework Preset**: `Vite`
- **Root Directory**: `client`（重要！指定前端目录）
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

**Environment Variables**（点击 "Environment Variables"）:

点击 "Add" 添加环境变量：

```
VITE_API_URL = https://researchlink-api.onrender.com/api
```

（使用上一步部署的后端 URL）

### 步骤 4.4：开始部署

1. 点击 "Deploy"
2. 等待构建完成（约 2-3 分钟）
3. 部署成功后，你会得到一个 URL，格式类似：
   ```
   https://researchlink.vercel.app
   ```
4. 保存这个 URL

### 步骤 4.5：更新后端 CORS 配置

1. 回到 Render Dashboard
2. 找到 `researchlink-api` 服务
3. 点击 "Environment" 标签
4. 找到 `CORS_ORIGIN` 变量
5. 修改值为：`https://researchlink.vercel.app`
6. 点击 "Save Changes"
7. Render 会自动重新部署

1. 访问 Vercel Dashboard
2. 选择 `researchlink` 项目

如果使用自定义域名，需要更新：

**Render 环境变量**：
```

```

1. 访问 MongoDB Atlas Dashboard
2. 点击 "Backups"

## ⚠️ 常见问题和解决方案

### 问题 1：Render 服务休眠

**现象**：首次访问 API 时响应很慢（10-30 秒）

**原因**：Render 免费层会在 15 分钟无请求后休眠服务

**解决方案**：
- 方案 A：升级到付费计划（$7/月）
- 方案 B：使用 UptimeRobot 每 14 分钟 ping 一次 API
- 方案 C：接受这个限制（个人项目足够）

### 问题 2：CORS 错误

**现象**：浏览器控制台显示 CORS policy 错误

**解决方案**：
1. 检查 Render 的 `CORS_ORIGIN` 环境变量是否正确
2. 确保格式为 `https://your-domain.vercel.app`（不要带斜杠）
3. 重新部署后端服务

### 问题 3：数据库连接失败

**现象**：API 返回 500 错误，日志显示 MongoDB 连接失败

**解决方案**：
1. 检查 MongoDB Atlas 的网络访问配置（确保添加了 0.0.0.0/0）
2. 检查连接字符串是否正确
3. 检查数据库用户名密码是否正确
4. 在 MongoDB Atlas 查看集群是否正常运行

### 问题 4：文件上传失败

**现象**：上传 Excel 文件时失败

**解决方案**：
1. 检查 Render 的文件大小限制（免费层 100MB）
2. 检查 `MAX_FILE_SIZE` 环境变量
3. 查看后端日志确认错误信息

### 问题 5：前端无法访问后端 API

**现象**：Network 标签显示 API 请求失败

**解决方案**：
1. 检查 Vercel 的 `VITE_API_URL` 环境变量
2. 确保 URL 格式正确（包含 `/api` 后缀）
3. 在浏览器中直接访问 API URL 测试
