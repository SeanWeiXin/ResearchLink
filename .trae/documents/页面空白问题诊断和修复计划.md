# 页面空白问题诊断和修复计划

## 问题描述
用户反馈页面没有任何内容显示，访问 http://localhost:5173 时页面空白。

## 可能原因分析

### 1. 后端数据库连接问题
- MongoDB 可能未启动或连接失败
- 数据库连接字符串配置错误
- 服务器启动但未正确初始化

### 2. 前端编译/运行时错误
- React 组件可能有未捕获的错误
- API 调用失败导致页面崩溃
- 缺少必要的依赖

### 3. API 通信问题
- 前后端端口不匹配
- CORS 配置问题
- API 路径配置错误

### 4. 认证状态问题
- localStorage 中的 token 无效
- 认证状态管理有误

## 诊断步骤

### 第一步：检查后端状态
1. 检查后端服务器是否正常启动
2. 查看后端终端是否有错误日志
3. 测试数据库连接
4. 访问 http://localhost:5000/health 检查健康状态

### 第二步：检查前端状态
1. 查看前端终端是否有编译错误
2. 打开浏览器开发者工具（F12）
3. 查看 Console 中的错误信息
4. 查看 Network 中的 API 请求状态

### 第三步：检查 API 通信
1. 确认 VITE_API_URL 配置正确
2. 测试 API 接口是否可访问
3. 检查 CORS 配置

### 第四步：检查数据流
1. 查看认证状态是否正确
2. 查看 API 请求是否发送
3. 查看响应数据是否正确

## 修复方案

### 方案 A：如果是后端问题

#### A1. 确保 MongoDB 正在运行
```powershell
# Windows 检查 MongoDB 服务
net start MongoDB

# 或者使用 MongoDB Compass 测试连接
```

#### A2. 检查后端配置
- 确认 `server/.env` 文件存在且配置正确
- 确认 MONGODB_URI 可连接
- 考虑使用 MongoDB Atlas 云数据库

#### A3. 重启后端服务
```bash
cd server
# 停止当前服务（Ctrl+C）
npm run dev
```

### 方案 B：如果是前端问题

#### B1. 检查前端配置
- 确认 `client/.env` 文件存在
- 确认 `VITE_API_URL=http://localhost:5000/api`
- 重启 Vite 开发服务器

#### B2. 清除缓存
```bash
cd client
# 删除 node_modules 和 lock 文件
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
# 重新安装
npm install
# 重新启动
npm run dev
```

#### B3. 添加错误边界
在 App.tsx 中添加错误处理，防止未捕获错误导致页面空白。

### 方案 C：如果是 API 通信问题

#### C1. 检查 API 客户端配置
确认 `client/src/api/client.ts` 中的 baseURL 正确。

#### C2. 添加错误处理
在所有 API 调用中添加 try-catch 和错误提示。

#### C3. 测试 API 连通性
使用 curl 或 Postman 测试 API 是否可访问。

### 方案 D：添加降级显示

#### D1. 添加加载状态
在页面加载时显示友好的提示信息。

#### D2. 添加错误提示
当 API 调用失败时，显示错误信息而不是空白页面。

#### D3. 添加空状态处理
当没有数据时，显示友好的空状态提示。
