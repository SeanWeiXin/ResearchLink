# ResearchLink 部署到互联网计划

## 项目分析

ResearchLink 是一个前后端分离的应用：
- **后端**：Node.js + Express + MongoDB（运行在端口 5000）
- **前端**：React + Vite（开发环境运行在端口 5173）
- **数据库**：MongoDB（本地连接：mongodb://localhost:27017/researchlink）

## 部署方案选择

### 方案一：全栈云平台部署（推荐初学者）

**平台组合**：
- 前端：Vercel / Netlify（免费，简单）
- 后端：Render / Railway（有免费层）
- 数据库：MongoDB Atlas（免费层 512MB）

**优点**：
- 配置简单，文档完善
- 免费层足够个人项目使用
- 自动 HTTPS 和 CDN
- 持续集成/部署

**缺点**：
- 免费层有性能限制
- 数据存储在第三方

### 方案二：传统 VPS 部署（推荐生产环境）

**平台**：
- 阿里云/腾讯云/华为云（国内）
- DigitalOcean / Linode / AWS（国外）

**优点**：
- 完全控制权
- 性能更好
- 数据自主可控

**缺点**：
- 需要运维知识
- 需要付费（约 50-100 元/月）
- 需要自己配置安全

### 方案三：Docker 容器化部署（推荐进阶用户）

**平台**：
- 任何支持 Docker 的云平台
- Kubernetes 集群

**优点**：
- 环境一致性
- 易于扩展
- 便于 CI/CD

**缺点**：
- 学习曲线陡峭
- 配置复杂

## 详细实施步骤

### 步骤 1：创建生产环境配置

创建 `server/.env.production`：
```env
PORT=5000
NODE_ENV=production
MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/researchlink
JWT_SECRET=<strong-random-secret>
MAX_FILE_SIZE=52428800
UPLOAD_PATH=./uploads
CORS_ORIGIN=https://your-domain.com
```

创建 `client/.env.production`：
```env
VITE_API_URL=https://api.your-domain.com/api
```

### 步骤 2：更新后端 CORS 配置

修改 `server/server.js` 支持生产环境域名。

### 步骤 3：配置 MongoDB Atlas

1. 访问 https://cloud.mongodb.com
2. 创建免费集群（M0）
3. 配置网络访问（添加 0.0.0.0/0）
4. 创建数据库用户
5. 获取连接字符串

### 步骤 4：部署后端到 Render

1. 注册 https://render.com
2. 创建新的 Web Service
3. 连接 GitHub 仓库
4. 配置环境变量
5. 部署命令：`cd server && npm install && npm start`
6. 获取部署 URL

### 步骤 5：部署前端到 Vercel

1. 注册 https://vercel.com
2. 导入 GitHub 仓库
3. 配置构建设置：
   - Root Directory: `client`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. 配置环境变量：`VITE_API_URL`
5. 部署

### 步骤 6：数据迁移（可选）

如果需要迁移现有数据：
```bash
# 导出本地数据
mongodump --uri="mongodb://localhost:27017/researchlink"

# 导入到 Atlas
mongorestore --uri="mongodb+srv://<user>:<password>@cluster.mongodb.net/researchlink"
```

## 备选方案：VPS 部署（简要）

如果选择 VPS 部署，步骤如下：

1. **购买服务器**
   - 选择 CentOS/Ubuntu 系统
   - 开放端口：80, 443, 22

2. **安装运行环境**
   ```bash
   # 安装 Node.js
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   
   # 安装 MongoDB
   sudo apt-get install -y mongodb-org
   
   # 安装 Nginx
   sudo apt-get install -y nginx
   ```

3. **部署应用**
   ```bash
   # 克隆代码
   git clone <repository>
   
   # 安装依赖
   cd server && npm install
   cd ../client && npm install
   
   # 构建前端
   cd client && npm run build
   ```

4. **配置 PM2 进程管理**
   ```bash
   npm install -g pm2
   pm2 start server/server.js --name researchlink-api
   pm2 save
   pm2 startup
   ```

5. **配置 Nginx 反向代理**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location /api {
           proxy_pass http://localhost:5000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
       
       location / {
           root /path/to/client/dist;
           try_files $uri $uri/ /index.html;
       }
   }
   ```

6. **配置 SSL 证书**
   ```bash
   sudo apt-get install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```
