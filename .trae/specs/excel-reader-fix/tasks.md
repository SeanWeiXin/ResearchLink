# Tasks

## 第一阶段：基础功能（2-3 天）

- [ ] Task 1: 修改后端上传 API，集成 Excel 解析服务
  - [ ] Task 1.1: 在 `uploads.js` 的 POST /excel 路由中调用 `processExcelFile` 函数
  - [ ] Task 1.2: 将解析结果保存到 Upload 模型的 metadata 字段
  - [ ] Task 1.3: 更新上传状态为'processed'（如果解析成功）
  - [ ] Task 1.4: 在响应中返回完整的元数据信息
  - [ ] Task 1.5: 添加错误处理，解析失败时设置状态为'error'并保存错误信息

- [ ] Task 2: 改进 Excel 解析服务（规则数据）
  - [ ] Task 2.1: 增强日期格式检测，支持更多日期格式（如 YYYY.MM.DD, DD/MM/YYYY 等）
  - [ ] Task 2.2: 改进数字检测，支持百分比、科学计数法等格式
  - [ ] Task 2.3: 优化列类型统计算法，样本数量从 5 提升到 20-50
  - [ ] Task 2.4: 添加数据质量预检查（空行、空列检测）
  - [ ] Task 2.5: 处理边界情况（空工作表、只有标题行等）

- [ ] Task 3: 实现数据预览功能
  - [ ] Task 3.1: 修改 `UploadPreview.tsx`，显示实际数据表格
  - [ ] Task 3.2: 从后端 API 获取数据样本（前 20 行）
  - [ ] Task 3.3: 使用 Ant Design Table 组件渲染数据
  - [ ] Task 3.4: 为每列添加类型标签和格式化显示
  - [ ] Task 3.5: 处理空值和特殊值的显示

## 第二阶段：智能识别（5-7 天）

- [ ] Task 4: 智能表头检测
  - [ ] Task 4.1: 实现空行检测算法（连续空行作为数据区域边界）
  - [ ] Task 4.2: 实现数据类型突变检测（文本→数字、数字→日期等）
  - [ ] Task 4.3: 检测合并单元格（使用 xlsx 库的 merges 属性）
  - [ ] Task 4.4: 实现多级表头列名合并逻辑
  - [ ] Task 4.5: 计算表头检测置信度评分

- [ ] Task 5: 数据区域分割
  - [ ] Task 5.1: 实现数据密度检测（每行/列的非空单元格比例）
  - [ ] Task 5.2: 识别独立数据区域（基于空行空列分割）
  - [ ] Task 5.3: 为每个区域生成唯一标识和边界坐标
  - [ ] Task 5.4: 实现区域选择策略（最大区域优先、用户选择等）
  - [ ] Task 5.5: 提供区域合并建议（当多个区域结构相似时）

- [ ] Task 6: 噪音过滤
  - [ ] Task 6.1: 检测并排除图表对象（使用 xlsx 的 drawings 属性）
  - [ ] Task 6.2: 识别汇总行关键词（"合计"、"总计"、"Total"、"Average"等）
  - [ ] Task 6.3: 检测注释文本框和批注
  - [ ] Task 6.4: 实现噪音行标记和过滤
  - [ ] Task 6.5: 记录被过滤内容的日志

- [ ] Task 7: 数据结构智能推断
  - [ ] Task 7.1: 实现时间序列检测（单一日期列 + 数值列）
  - [ ] Task 7.2: 实现面板数据检测（个体列 + 时间列 + 指标列）
  - [ ] Task 7.3: 实现截面数据检测（单时间点 + 多个体）
  - [ ] Task 7.4: 为每种数据类型生成处理建议
  - [ ] Task 7.5: 提供面板数据转时间序列的筛选建议

- [ ] Task 8: 多工作表处理策略
  - [ ] Task 8.1: 比较各工作表结构相似性（列名、列数、数据类型）
  - [ ] Task 8.2: 实现工作表合并算法（添加"来源表"列）
  - [ ] Task 8.3: 提供工作表选择 UI（单选、多选、全选）
  - [ ] Task 8.4: 为相似工作表提供合并建议
  - [ ] Task 8.5: 处理合并后的数据一致性问题

- [ ] Task 9: 置信度评估系统
  - [ ] Task 9.1: 设计置信度评分算法（表头清晰度、数据连续性、类型一致性、噪音比例）
  - [ ] Task 9.2: 实现置信度计算逻辑
  - [ ] Task 9.3: 根据置信度分级处理（高/中/低）
  - [ ] Task 9.4: 在响应中返回置信度和警告信息
  - [ ] Task 9.5: 在前端显示置信度标签和提示

- [ ] Task 10: 人工校正界面
  - [ ] Task 10.1: 设计校正界面 UI（数据范围选择器、表头指定、区域框选）
  - [ ] Task 10.2: 实现数据起始/结束行选择器
  - [ ] Task 10.3: 实现表头行指定功能
  - [ ] Task 10.4: 实现可视化区域框选（类似 Excel 选择区域）
  - [ ] Task 10.5: 实现噪音行标记/取消标记功能
  - [ ] Task 10.6: 实时预览选择的数据
  - [ ] Task 10.7: 保存用户校正结果到数据库

- [ ] Task 11: 前端智能提示
  - [ ] Task 11.1: 显示数据结构类型标签（时间序列/面板/截面）
  - [ ] Task 11.2: 智能推荐列选择（时间列、个体列、指标列）
  - [ ] Task 11.3: 显示推荐理由和置信度提示
  - [ ] Task 11.4: 提供数据转换建议（如面板转时间序列）
  - [ ] Task 11.5: 警告和错误信息的友好展示

## 第三阶段：测试和优化（2-3 天）

- [ ] Task 12: 测试和验证
  - [ ] Task 12.1: 收集真实复杂 Excel 样本（多行表头、多区域、面板数据等）
  - [ ] Task 12.2: 测试各种格式的 Excel 文件（.xlsx, .xls, .csv）
  - [ ] Task 12.3: 测试边界情况（空文件、超大文件、损坏文件）
  - [ ] Task 12.4: 验证智能识别的准确率
  - [ ] Task 12.5: 性能优化（大文件解析时间、内存占用）
