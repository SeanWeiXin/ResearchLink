# Excel 数据读取功能重构 Spec - 智能化数据处理

## Why
当前 Excel 读取功能几乎无法使用，主要问题是：
1. 上传后没有实际调用 Excel 解析服务，文件只是保存到磁盘，没有提取元数据
2. 数据预览功能显示"开发中..."，用户无法看到实际数据
3. 缺少对解析失败的处理和错误提示
4. 列类型检测不够准确，特别是日期和数字的识别

**更关键的是**，用户上传的 Excel 可能存在复杂格式：
- 表头行数不固定（1 行到多行合并单元格）
- 表头位置不固定（可能从第 3 行、第 5 行开始）
- 同一工作表包含多个独立数据区域
- 同时存在时间序列和面板数据
- 工作表中夹杂图表、注释、汇总等噪音数据
- 多个工作表包含不同结构的数据

## What Changes
### 第一阶段：基础功能（解决规则数据）
- **新增**：在上传 API 中集成 Excel 解析服务，上传时自动提取元数据
- **新增**：实现完整的数据预览功能，显示实际数据内容
- **新增**：添加解析错误处理和用户友好的错误提示
- **改进**：优化列类型检测算法，提高日期和数字识别准确率
- **改进**：支持更多日期格式和数字格式的检测

### 第二阶段：智能识别（解决复杂格式）
- **新增**：智能表头检测 - 自动识别表头行位置（基于数据类型突变、空行等特征）
- **新增**：数据区域分割 - 识别并分离同一工作表中的多个独立数据块
- **新增**：噪音过滤 - 自动排除图表、注释、汇总行等非数据行
- **新增**：数据结构推断 - 智能识别时间序列、面板数据、截面数据
- **新增**：多工作表策略 - 提供工作表选择建议或合并策略
- **新增**：人工校正界面 - 当自动识别置信度低时，允许用户手动指定数据区域

## Impact
- 受影响的服务：`excelProcessor.ts`, `uploads.js`
- 受影响的前端组件：`UploadExcel.tsx`, `UploadPreview.tsx`
- 受影响的 API：`POST /api/uploads/excel`, `GET /api/uploads/:id`

## ADDED Requirements

### Requirement: 基础解析功能（第一阶段）
系统 SHALL 正确解析规则格式的 Excel 文件：
- 表头在第一行
- 数据连续排列，没有空行
- 单一数据区域
- 返回元数据：列名、列类型、行数、列数、工作表列表

#### Scenario: 规则数据解析成功
- **WHEN** 用户上传标准格式的 Excel 文件
- **THEN** 系统应在 10 秒内完成解析
- **THEN** 正确识别所有列的类型（日期、数字、文本）
- **THEN** 在预览页面显示前 20 行数据

### Requirement: 智能表头检测（第二阶段）
系统 SHALL 自动检测表头的实际位置：
- 检测空行作为数据分隔符
- 检测数据类型突变（如从文本突然变为数字）
- 检测合并单元格
- 支持 1-5 行表头

#### Scenario: 多行表头识别
- **WHEN** Excel 文件前 3 行是表头（含合并单元格）
- **THEN** 系统应识别第 4 行为数据起始行
- **THEN** 正确合并多级表头的列名（如"销售额 - 第一季度"）

### Requirement: 数据区域分割（第二阶段）
系统 SHALL 识别并分离同一工作表中的多个独立数据区域：
- 检测空行和空列作为区域边界
- 检测数据密度突变
- 为每个独立区域生成独立的数据块标识
- 提供区域选择或合并建议

#### Scenario: 多区域数据识别
- **WHEN** 工作表中包含 2 个独立数据表（中间有空行分隔）
- **THEN** 系统应识别出 2 个独立数据区域
- **THEN** 提供选择：分别处理 或 合并处理
- **THEN** 显示每个区域的行数、列数、数据结构

### Requirement: 噪音过滤（第二阶段）
系统 SHALL 自动识别并排除非数据内容：
- 图表对象（Excel 图表、图片）
- 注释文本框
- 汇总行（包含"合计"、"总计"、"Average"等关键词）
- 页眉页脚
- 签名、日期戳等元数据

#### Scenario: 噪音数据过滤
- **WHEN** 工作表底部包含"合计"行和图表
- **THEN** 系统应排除这些行，不纳入数据预览
- **THEN** 在日志中标注被过滤的内容及原因

### Requirement: 数据结构智能推断（第二阶段）
系统 SHALL 推断数据的整体结构类型：
- **时间序列数据**：单一个体随时间变化
- **面板数据**：多个体随时间变化（如多省份多年份）
- **截面数据**：单一时间点的多个个体
- **其他结构**：无法归类时标记为"自定义"

#### Scenario: 面板数据识别
- **WHEN** 数据包含"省份"、"年份"和多指标列
- **THEN** 系统应识别为面板数据
- **THEN** 建议：选择个体列（省份）、时间列（年份）、指标列（数值）
- **THEN** 提示用户是否需要转换为时间序列（筛选特定省份）

### Requirement: 多工作表处理策略（第二阶段）
系统 SHALL 为多工作表提供处理建议：
- 检测各工作表结构是否相似
- 提供选项：分别处理、合并处理、选择主表
- 对于相似结构的工作表，建议合并（如"2020 年"、"2021 年"、"2022 年"）
- 对于不同结构的工作表，建议分别处理

#### Scenario: 多工作表合并
- **WHEN** Excel 包含 3 个结构相同的工作表（年份不同）
- **THEN** 系统应建议合并处理
- **THEN** 自动添加"来源工作表"列以区分数据
- **THEN** 合并后显示总行数和列数

### Requirement: 人工校正界面（第二阶段）
系统 SHALL 在自动识别置信度低时提供人工校正界面：
- 置信度 < 70% 时触发人工校正
- 允许用户手动指定：数据起始行、数据结束行、表头行
- 允许用户框选数据区域（可视化选择）
- 允许用户标记/取消标记噪音行
- 实时预览选择的数据

#### Scenario: 人工校正数据区域
- **WHEN** 系统无法确定表头位置（置信度 50%）
- **THEN** 显示校正界面，高亮显示推测的表头行
- **THEN** 用户可以拖动边界调整数据范围
- **THEN** 实时显示选中数据的预览
- **THEN** 用户确认后保存到数据库

### Requirement: 解析置信度评估（第二阶段）
系统 SHALL 为每次解析生成置信度评分（0-100%）：
- 基于：表头清晰度、数据连续性、类型一致性、噪音比例
- 置信度 >= 80%：自动通过，显示"高置信度"标签
- 置信度 50%-79%：建议用户检查，显示"中等置信度"标签
- 置信度 < 50%：强制人工校正，显示"低置信度 - 需要人工确认"

#### Scenario: 置信度提示
- **WHEN** 解析完成后置信度为 65%
- **THEN** 在预览页面显示警告提示
- **THEN** 建议用户检查表头位置和数据范围
- **THEN** 提供"重新选择数据区域"按钮

## MODIFIED Requirements

### Requirement: 上传 API 响应
**原要求**：上传成功后只返回基本文件信息
**新要求**：上传成功后应返回完整的元数据信息和置信度评估：
```json
{
  "upload": {
    "id": "...",
    "filename": "...",
    "status": "processed",
    "metadata": {
      "rowCount": 100,
      "columnCount": 5,
      "sheetNames": ["Sheet1"],
      "columns": [
        { "name": "日期", "type": "date" },
        { "name": "销售额", "type": "number" }
      ],
      "isTimeSeries": true,
      "timeSeriesFormat": "YYYY-MM-DD",
      "dataType": "time_series",
      "confidence": 0.92,
      "dataRegions": [{
        "startRow": 3,
        "endRow": 102,
        "headerRows": [0, 1, 2],
        "isClean": true
      }]
    },
    "warnings": ["检测到 2 个独立数据区域，已自动选择最大的区域"]
  }
}
```

### Requirement: 预览页面数据加载
**原要求**：预览页面需要手动配置后才能看到数据
**新要求**：预览页面应：
1. 自动显示解析后的数据预览（前 20 行）
2. 显示置信度评分和警告信息
3. 对于低置信度数据，提供人工校正入口
4. 支持切换不同的数据区域（如果检测到多个）
5. 显示数据结构类型提示（时间序列/面板数据/截面数据）

### Requirement: 数据处理配置
**原要求**：用户手动选择日期列和数值列
**新要求**：智能推荐 + 人工确认：
1. 系统自动推荐时间列、个体列（面板数据）、指标列
2. 显示推荐理由（如"该列包含日期格式数据"）
3. 用户可修改推荐结果
4. 对于面板数据，提供转换选项（筛选特定个体转为时间序列）

## REMOVED Requirements

### Requirement: 手动触发解析
**原因**：上传后自动解析，无需手动触发
**迁移**：删除相关的手动解析按钮和 API
