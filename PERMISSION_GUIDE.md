# 权限管理系统使用指南

## 📋 概述

ResearchLink 系统现在包含完整的权限管理功能，管理员可以通过 Web 界面管理用户角色和权限。

## 👥 用户角色

系统支持三种用户角色：

| 角色 | 显示名称 | 说明 |
|------|---------|------|
| `user` | **探索者** | 普通用户，可以浏览、发帖、评论、点赞、收藏 |
| `uploader` | **贡献者** | 可以上传 Excel 文件、创建和配置图表 |
| `admin` | **维护者** | 管理员，拥有所有权限，可以管理其他用户 |

## 🔐 权限类型

| 权限 | 说明 |
|------|------|
| 上传 Excel | 允许用户上传和解析 Excel 数据文件 |
| 创建图表 | 允许用户根据数据生成可视化图表 |
| 发布图表 | 允许用户将图表发布到论坛帖子中 |

## 🚀 快速开始

### 1. 设置第一个管理员

**方法一：使用脚本（推荐）**

```bash
# 在项目根目录执行
node scripts/setup-admin.js
```

按照提示输入用户名，该用户将被设置为管理员。

**方法二：直接修改数据库**

使用 MongoDB Compass 或命令行：

```javascript
use researchlink;
db.users.updateOne(
  { username: "your_username" },
  { 
    $set: { 
      role: "admin",
      "permissions.canUploadExcel": true,
      "permissions.canCreateCharts": true,
      "permissions.canPublishCharts": true
    }
  }
)
```

### 2. 登录管理员账户

1. 访问 http://localhost:5173
2. 使用管理员账户登录
3. 页面右上角会出现 **"管理后台"** 按钮

### 3. 访问管理后台

点击 **"管理后台"** 按钮或直接访问 http://localhost:5173/admin

## 🎛️ 管理后台功能

### 用户列表

- 查看所有注册用户
- 显示用户头像、用户名、邮箱
- 显示当前角色（带颜色标签）
- 显示拥有的权限

### 编辑用户

点击用户行的 **"编辑"** 按钮可以：

1. **修改角色**
   - 探索者（user）
   - 贡献者（uploader）
   - 维护者（admin）

2. **配置权限**
   - ✓ 上传 Excel - 允许上传和解析 Excel 文件
   - ✓ 创建图表 - 允许生成可视化图表
   - ✓ 发布图表 - 允许发布图表到论坛

3. **查看权限说明**
   - 底部有详细的权限说明文本

### 权限标签显示

在个人中心页面，用户可以看到自己的角色和权限：

- **角色标签**：蓝色背景，显示"探索者"/"贡献者"/"维护者"
- **权限标签**：
  - 上传 - 绿色
  - 图表 - 青色
  - 发布 - 紫色

## 📱 页面位置

### 个人中心
- 路径：`/profile`
- 显示当前用户的角色和权限
- 可以编辑个人资料、查看帖子和收藏

### 管理后台
- 路径：`/admin`
- 仅管理员可访问
- 管理所有用户的角色和权限

## 🔒 安全保护

### 路由保护

- `/admin` 路由有权限检查
- 非管理员访问会自动重定向到首页
- 未登录用户会被重定向到登录页

### API 保护

后端 API 有中间件验证：
- 验证 JWT token
- 检查用户角色
- 非管理员请求返回 403 错误

## 📊 使用场景

### 场景 1：授予上传权限

1. 管理员登录
2. 访问管理后台
3. 找到目标用户
4. 点击"编辑"
5. 开启"上传 Excel"权限
6. 保存

### 场景 2：提升为贡献者

1. 管理员登录
2. 访问管理后台
3. 编辑活跃用户
4. 角色改为"贡献者"
5. 自动获得上传和创建图表权限

### 场景 3：任命管理员

1. 现有管理员登录
2. 访问管理后台
3. 编辑信任的用户
4. 角色改为"维护者"
5. 该用户获得所有管理权限

## 🛠️ 技术实现

### 后端 API

```javascript
// 获取所有用户
GET /api/auth/admin/users
Headers: Authorization: Bearer <token>
要求：admin 角色

// 更新用户信息
PUT /api/auth/admin/users/:id
Headers: Authorization: Bearer <token>
Body: {
  role: "admin" | "uploader" | "user",
  permissions: {
    canUploadExcel: boolean,
    canCreateCharts: boolean,
    canPublishCharts: boolean
  }
}
要求：admin 角色
```

### 前端组件

- `Admin.tsx` - 管理后台页面
- `admin.ts` - 管理 API 客户端
- `AdminRoute` - 管理员路由保护组件

## 💡 最佳实践

1. **最小权限原则**
   - 只授予必要的权限
   - 定期审查用户权限

2. **管理员数量**
   - 建议保持 2-3 个管理员
   - 避免权限过于集中

3. **贡献者管理**
   - 根据活跃度授予贡献者身份
   - 鼓励用户上传和分享数据

4. **权限审计**
   - 定期检查用户角色分布
   - 移除不再需要的权限

## ❓ 常见问题

### Q: 忘记所有管理员账户怎么办？

A: 直接在数据库中修改：
```javascript
db.users.updateOne(
  { username: "your_username" },
  { $set: { role: "admin" } }
)
```

### Q: 可以删除用户吗？

A: 当前版本不支持删除功能。如需删除，请使用 MongoDB 工具。

### Q: 权限修改后多久生效？

A: 立即生效。用户刷新页面即可看到新权限。

### Q: 管理员可以修改其他管理员吗？

A: 可以，管理员可以修改任何非管理员用户的角色。

### Q: 如何撤销权限？

A: 在管理后台编辑用户，关闭相应的权限开关即可。
